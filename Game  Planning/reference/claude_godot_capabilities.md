# Claude × Godot 対応範囲
> 最終更新: 2026-03-28

---

## ✅ Claudeが直接できること

### スクリプト（.gd）
- 新規作成・全面書き換え・部分編集
- バグ修正・リファクタリング
- 新機能の追加
- autoload（シングルトン）の作成・更新

### シーン（.tscn）
- 新規作成
- ノードの追加・削除・プロパティ変更
- スクリプトのアタッチ・変更
- ノード間のシグナル接続（`[connection]`）
- `y_sort_enabled` などのノードプロパティ設定
- `MultiplayerSpawner` の `_spawnable_scenes` 更新

### プロジェクト設定（project.godot）
- オートロードの追加・削除・順序変更
- 入力アクション（InputMap）の追加・編集
- レンダラーや画面サイズなどの設定変更

### リソース（.tres）
- テキスト形式のものは読み書き可能
- アニメーションデータ（AnimationLibrary）
- シェーダー（.gdshader）

### ファイル管理
- ディレクトリの作成
- ファイルの移動・名前変更（Bash経由）
- 不要ファイルの削除

### Git
- コミット・プッシュ
- ブランチ操作

---

## ❌ Claudeにできないこと（エディターが必要）

### ビジュアル編集
- TileSetのタイル登録・アトラス設定
- アイソメトリックTileMapのタイル形状変更
- AnimationPlayerのアニメーションキーフレーム編集
- ShaderMaterialのビジュアルシェーダー編集
- Spriteのテクスチャ確認・トリミング

### アセットのインポート
- 画像・音声・フォントのインポート設定（`.import`ファイル）
- 新しいアセットをエディターに認識させる操作

### 実行・テスト
- ゲームを実際に起動して動作確認
- ランタイムエラーのデバッグ（出力ログは共有してもらえれば対応可）
- パフォーマンスプロファイリング

### 複雑なリソース
- PackedScene内の深いネスト構造（エディターで編集した方が安全）
- NavigationMesh・NavigationRegionの設定
- PhysicsMaterialなどの物理リソース

## 🔄 連携のベストプラクティス

| やること | 誰が担当 |
|---|---|
| スクリプトのロジック実装 | Claude |
| シーン構造の設計・作成 | Claude |
| TileSet・アニメーション設定 | エディター（手動） |
| 実行してエラーが出たらログを共有 | Claude が解析・修正 |
| アセット（画像・音声）の追加 | エディター（手動インポート） |
| Git管理 | Claude |

---

## 💡 エラー対応の流れ

Godotエディターでエラーが出た場合：

1. エディター下部の**デバッグコンソール**のエラーメッセージをコピー
2. Claudeに貼り付けて共有
3. Claudeが原因を特定して該当ファイルを修正
4. エディターがファイルの変更を自動検知して再読み込み
